Question: 


      >>> if "":
      ...     return 1/0
      ... else:
      ...     return 0/1

Answer: 


       0.0

Notice:
(1) Remember that "" is a False-y value in Python. Thus, the 1/0 does not get evaluated because the True suite is not entered. (Check: What are the five False-y values in Python?)
(2) Notice that the return value is the evaluated value (i.e. 0.0) of the expression, and not the expression itself (i.e. 1/0). (Check: What are expressions and values?)


Question: 


      >>> if "False":
      ...     print("False")
      ... else:
      ...     print("True")

Answer: 


       False
       True

Notice:
(1) Note that "False" is not the same thing as False. "False" is a non-empty string that evaluates to True, False is the False value in Python.
(2) Notice the difference between print vs. return. With print, we continue to evaluate the rest of the body until we reach a return. With return, we return the value and halt execution.
(3) Nitpick: When we use print on a string, the interpreter shows no quotes. When we use return on a string, the interpreter prints the quotes. (i.e. print("hello") becomes hello while return "hello" becomes 'hello')


Question: 


      >>> n = 0:
      ... while n < 3:
      ...     return 3
      ...     n += 1

Answer: 


       3

Notice:
(1) What is the difference between return vs. print in terms of what the next step is? Does the line n += 1 ever get reached here?


Question: 


      >>> n = 0:
      ... while n < 3:
      ...     print(3)

Answer: 


       (infinite loop)

Notice:
(1) Remember to change the variable such that the condition eventually evaluates to False and we exit the loop.


Question: 


      >>> n = 0:
      ... while n < 3:
      ...     n += 1
      ... return n

Answer: 


       3

Notice:
(1) Notice where the return statement is: since it is on the outside of the while loop, we return n only once we have exited the loop.
(2) Why is the return value 3 and not another number? When does the condition become False?
